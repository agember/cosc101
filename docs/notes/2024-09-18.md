## Collections

* _What are the three types of values we have seen/used in programs thus far?_ â€“ int, float, string
* Regardless of the number of digits in an integer, we treat it as a single value â€“ e.g., `7`, `99`, `1819`
    * Similarly for floats
    * Similarly for strings â€“ e.g., `"awesome"`, `"How are you?"`, `"13"`
* However, a string is actually a collection of "smaller" values â€“ individual characters
    * e.g., `"awesome"` is the sequence of characters a, w, e, s, o, m, e
    * `"a" + "w" + "e" + "s" + "o" + "m" + "e"` evaluates to "awesome"
    * We cannot think of integers, or floats, as a collection of values, because they don't combine in the same way â€“ e.g., `9 + 9` evalutes to `18` not `99`

## Lists

* Sequential collection of any type of values (e.g., int, float, string)
    * List may contain multiple types of values
    * List may even include another list â€” called a sublist
* Create a list using square brackets


```python
list1 = [10, 20, 'aniyah', 'bilal']
list2 = [] # empty list
list3 = ['carlos', [50, 60]]
```

* Each item in the list has an index â€” just like strings
    * The first (i.e., leftmost) item has index 0
    * Use negative indices to refer to items starting from the last (i.e., rightmost) item
    * Use the slice operator to access a subset of the items in the list
